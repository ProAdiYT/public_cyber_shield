# Contributing to Citizen Cyber Shield

Thank you for your interest in contributing to Citizen Cyber Shield! Your help is critical to building next-generation threat intelligence platforms.

## Code of Conduct
Please be respectful and professional in all interactions. We aim to foster a collaborative and welcoming open-source community.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create a GitHub Issue. Include:
* A clear description of the issue.
* Steps to reproduce the bug.
* Expected behavior vs. actual behavior.
* Environment details (OS, Python version, browser, etc.).

### Proposing Enhancements
If you have ideas for new features or scan models:
1. Open a GitHub Issue to discuss your idea.
2. Outline the benefits and technical approach.

### Submitting Pull Requests
1. Fork the repository.
2. Create a new branch: `git checkout -b feature/your-feature-name`.
3. Make your changes:
   * Keep changes focused and well-documented.
   * Write clean, commented code.
4. Verify your changes locally (run both the backend and frontend tests).
5. Commit and push: `git commit -m "Add feature details"` and push to your fork.
6. Open a Pull Request against the `main` branch.

## Development Checklist
* Run tests before submitting PRs.
* Document any changes to endpoints in `SETUP.md` or `README.md`.
* Ensure that large datasets are not committed (use the `.gitignore` exclusions).
